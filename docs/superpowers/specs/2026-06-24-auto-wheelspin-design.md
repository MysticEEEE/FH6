# 自动抽奖（Wheelspin）流程 设计文档

- 日期：2026-06-24
- 状态：待评审
- 作者：MysticEEEE（与 Claude 协作）
- 关联：与「自动送车」（`2026-06-24-auto-gift-cars-design.md`）为两个独立功能，各自 spec / 实现 / worktree。

## 1. 目标

新增「自动抽奖」流程：自动进入游戏的抽奖功能，连续抽奖；当抽到**已拥有的重复车**时，
自动在「已拥有车辆」对话框中选择「出售」直接卖掉，避免误加入库。一直抽到次数用尽。

支持两种模式（可配置）：**抽奖（Wheelspin）** 与 **超级抽奖（Super Wheelspin）**，
两者循环逻辑一致，仅入口与单抽产出数量不同（超级抽奖一次最多 3 车）。

## 2. 背景与现有可复用资产

沿用现有图像识别流程的写法与基础设施：

| 复用对象 | 位置 | 用途 |
|---|---|---|
| `enter_menu()` | `main.py:1800` | 进入主菜单（导航起点） |
| 导航模式 | `logic_race`/`logic_super_wheelspin` | `enter_menu()` → 定位锚点 → `game_click` 的模式 |
| 通用工具 | `game_click`/`hw_press`/`wait_for_image_gray`/`wait_for_any_image_gray`/`find_image_gray`/`capture_region`/`check_pause`/`update_running_ui` | 点击、按键、找图、截图、暂停、进度 |

**命名澄清**：现有 `logic_super_wheelspin`（`main.py:2518`）名为「超级抽奖」，
实为给车辆**点技能点/专精**，并非真正抽奖。本功能是**全新的真抽奖流程**，与之无关，
不复用其业务逻辑（仅可参考其导航写法）。

**AI/YOLO**：本流程的识别目标是 UI 对话框（「已拥有车辆」），非车卡全新标记，
用模板匹配即可，**不使用 YOLO，不需训练**。

## 3. 游戏流程观察（依据截图 12–18）

- 截图12：「我的地平线」菜单页，含「超级抽奖」(183 可用) 与「抽奖」(32 可用) 入口。
- 截图13/16：转盘动画界面，左下角「剩余抽奖机会」计数 + `Enter 跳过`。超级抽奖为 3 个转盘。
- 截图14/17：跳过后的奖励结果，左下「Enter 领取奖励并再次抽奖」/「Esc 领取奖励」。
- 截图15/18：抽到重复车时弹出「**已拥有车辆**」对话框，三个选项：
  1. 添加至车库（**默认高亮**）
  2. 送礼
  3. 出售价格 ⊙xx,xxx ← **目标选项**

  → 处理：`下`×2 移到「出售」→ `Enter` 卖出 → 回到转盘。
  **超级抽奖一次最多产出 3 辆车**，因此一次「领取」后这些重复对话框会**连续、依次弹出，最多 3 次**：
  第 1 次触发重复 → 选「出售」`Enter` 卖出 → **紧接着自动弹出**第 2 次（若该次也是重复）→ 再卖出 →
  最多到第 3 次。每一次都必须先卖出，下一次才会出现。所以处理逻辑必须**循环检测、逐个卖出，直到不再弹出对话框**
  （不能假设只有一次）。普通抽奖一次 1 车，最多 1 次。

## 4. 流程设计

### 4.1 导航进入

```
enter_menu()                          # 复用：进入主菜单
→ 定位「我的地平线」页（菜单锚点）
→ 按 wheelspin_mode 配置，game_click「抽奖」或「超级抽奖」入口
```

### 4.2 核心循环

```
循环（直到停止条件）：
  1. 转盘动画 → hw_press("enter") 跳过      # 动画不涉及车，Enter 安全
  2. 出现奖励结果 → 鼠标点击左下角「领取并再抽」按钮
       （首次用模板匹配定位按钮坐标，之后复用缓存坐标；不影响效率）
  3. 检测「已拥有车辆」对话框？
       有 → hw_press("down")×2 → hw_press("enter")（卖出重复车）
             → 回步骤 3 再检测（超抽连续多车，最多 3 次；逐个卖出直到无对话框）
       无 → 回步骤 1（下一抽）
  4. 停止判断（见 4.4）
```

### 4.3 安全闸门（最高优先级）

唯一红线：「已拥有车辆」对话框默认高亮在「添加至车库」（选项1）。
因此 **每次领取后必须先检测该对话框，检测到才用方向键+Enter 卖出，
绝不盲目连按 Enter**（盲按会落到选项1 → 误加入库）。

为进一步降险，**领取/再抽动作用鼠标点击左下角按钮**，而非 Enter，
避免连发的 Enter 在对话框出现时误触选项1。

### 4.4 停止条件（两者都支持）

1. **抽完为止**：领取并处理完对话框后，若画面**退回「我的地平线」菜单**
   （检测菜单锚点）或检测到剩余次数为 0 → 停止。
2. **次数上限**：可配置 `wheelspin_max_count`（0=不限），达上限即停。安全阀。

> 以「退回菜单」作为确定性结束信号优先；计数上限为安全阀。

## 5. 模块与边界

新增聚焦的流程方法（拟名 `logic_auto_wheelspin`），放入现有流程模块区。
职责单一：执行一次完整自动抽奖。辅助函数（按需拆分）：

- `navigate_to_wheelspin(mode)` — 导航并进入指定抽奖入口
- `skip_wheelspin_animation()` — Enter 跳过动画，等待结果就位
- `collect_and_respin()` — 鼠标点击「领取并再抽」推进
- `handle_owned_car_dialog()` — 检测「已拥有车辆」并卖出（循环处理多车），返回是否处理过
- `is_wheelspin_finished()` — 判断停止（退回菜单 / 计数耗尽）

## 6. 配置项（写入 config.json）

| 键 | 默认 | 含义 |
|---|---|---|
| `wheelspin_mode` | "抽奖" | 抽奖类型："抽奖" 或 "超级抽奖" |
| `wheelspin_max_count` | 0 | 最大抽奖次数上限（0=不限，抽完为止） |
| 任务链开关（复用现有 chk 机制或新增） | false | 是否纳入任务链 |

## 8. 接入方式

- **GUI 按钮（主要）**：界面新增「自动抽奖」按钮 + 模式选择（抽奖/超级抽奖）。
- **任务链（可选）**：提供可选配置项，无人值守时可纳入任务链。
- **不分配热键**。

## 9. 风险与缓解

| 风险 | 缓解 |
|---|---|
| 重复车误加入库 | 每次领取后先检测「已拥有车辆」对话框；领取用鼠标点击而非盲按 Enter |
| 超抽一次多车，漏处理 | 对话框处理为循环，直到检测不到对话框为止 |
| 左下按钮定位偏差 | 首次模板匹配定位 + 坐标缓存；定位失败回退到 Enter（仅在确认无对话框时） |
| 抽奖结束判定不准 | 以「退回我的地平线菜单」锚点为确定性结束信号 + 次数上限安全阀 |
| 动画/切换时序导致连点错位 | 关键节点用 `wait_for_image_gray` 等画面就位，不盲目连点 |

## 10. 范围之外（YAGNI）

- 不使用 YOLO / 不训练模型。
- 不做奖品价值判断（除「已拥有重复车→出售」外，其余奖励一律默认领取）。
- 不复用 `logic_super_wheelspin`（点技能点）的业务逻辑。
- 不同时自动跑「抽奖+超级抽奖」混合策略（V1 单模式可配置；如需两者都跑可分两次运行）。
